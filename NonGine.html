Intro
NonGine started of as a course we had at The Game Assembly where we designed and build a DirectX11 game engine from scratch to finsihed engine. I liked the idea of making a custom engine that i can then chose how it looks. I also decided to at around the middle of the course to make a quick editor that you can transofrm objects, add components and adjust parameters on added components to objects. The game engine has a Component System and a save and load system to save and load to a file.
An example of how NonGine in game editor looks
I have now after the Specialization course i partook in at TGA recreated the core parts of the engine in OpenGL to both improve performance and see how it is developing in a different graphics API Both the OpenGL and DX11 version of the engine has support for physx and physics objects. This has been one of my biggest projects and i'm going to continue improve the engine to make it more use friendly and optimised
